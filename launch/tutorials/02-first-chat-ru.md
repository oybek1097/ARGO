# Урок 02 — Первый разговор и работа с CLI (Russian)

**Тема:** Первый разговор и интерфейс командной строки
**Язык:** Русский
**Целевая длительность:** ~5 минут

---

### Сцена 1 — Вступление (0:00–0:25)

ON SCREEN: логотип ARGO, затем терминал.

NARRATION: «С возвращением. В прошлом видео мы установили ARGO. Теперь
давайте им воспользуемся — проведём первый разговор, посмотрим на
интерфейс командной строки и увидим, как ARGO определяет языки.»

### Сцена 2 — Обзор CLI (0:25–1:05)

ON SCREEN:
```
python3 -m argo_brain --help
```

ON SCREEN (выделить перечисленные команды):
```
setup     doctor    chat      tui
serve     ipc       telegram  mcp
selftest  version
```

NARRATION: «У ARGO одна точка входа — модуль argo_brain — с несколькими
подкомандами. Команды setup и doctor мы уже использовали. Сегодня
главная — chat. Есть также serve для HTTP-шлюза, telegram для
Telegram-бота и tui для расширенного терминального интерфейса.»

### Сцена 3 — Запуск чата (1:05–1:45)

ON SCREEN:
```
python3 -m argo_brain chat
```

NARRATION: «Выполните команду chat. Она откроет интерактивную сессию. По
умолчанию используется локальная тестовая модель, поэтому ключ API не
нужен — это идеально, чтобы изучить поведение агента. Поздороваемся.»

ON SCREEN (вводит пользователь):
```
> Hello, what can you do?
```

NARRATION: «ARGO отвечает, и вы можете продолжать разговор.»

### Сцена 4 — Многоязычный разговор (1:45–2:55)

ON SCREEN (вводит пользователь):
```
> Salom! Bugun ob-havo qanday?
```

NARRATION: «А вот то, ради чего ARGO создан. Я написал это на узбекском.
ARGO определяет язык и возвращает ответ на том же языке. Попробуем и
русский.»

ON SCREEN (вводит пользователь):
```
> Привет! Расскажи о себе.
```

NARRATION: «С русским всё работает так же. ARGO поддерживает узбекский,
русский, казахский, киргизский, таджикский и английский — определение
языка встроено, а не добавлено сбоку.»

### Сцена 5 — Цикл агента и инструменты (2:55–4:00)

ON SCREEN (вводит пользователь):
```
> What files are in the current directory?
```

NARRATION: «ARGO — это не просто чат-бот, это агент. Когда для запроса
нужно действие, ARGO планирует, затем выполняет инструмент, а затем
использует результат. Это цикл «планирование, затем выполнение». Сегодня
ARGO поставляется с 13 встроенными инструментами, включая оболочку,
файлы, Git и Docker.»

ON SCREEN: агент показывает шаг с инструментом, затем ответ.

NARRATION: «Здесь виден шаг с инструментом, а затем итоговый ответ,
построенный из результата.»

### Сцена 6 — Память (4:00–4:35)

ON SCREEN (вводит пользователь):
```
> My name is Aziz.
> What is my name?
```

NARRATION: «ARGO запоминает контекст в рамках разговора. Он хранит
кратковременную память в процессе и долговременное хранилище в локальной
базе данных SQLite на вашем диске — ваша история никогда не покидает ваш
компьютер.»

### Сцена 7 — Заключение и призыв к действию (4:35–5:00)

ON SCREEN: логотип ARGO, адрес GitHub, надпись «Далее: подключение
Telegram-бота».

NARRATION: «Вот ваш первый разговор с ARGO: CLI, многоязычные ответы,
инструменты и память. ARGO — открытый и распространяется под лицензией
MIT, сейчас он в стадии альфа на пути к общему релизу — попробуйте его и
расскажите, что сломалось. В следующем видео мы подключим ARGO к
Telegram-боту. Спасибо за просмотр.»
