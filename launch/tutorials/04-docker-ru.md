# Урок 04 — Развёртывание с Docker Compose (Russian)

**Тема:** Самостоятельный хостинг ARGO с Docker Compose
**Язык:** Русский
**Целевая длительность:** ~5 минут

---

### Сцена 1 — Вступление (0:00–0:25)

ON SCREEN: логотип ARGO, затем терминал.

NARRATION: «Добро пожаловать в финальное видео этой серии. Мы установили
ARGO, пообщались с ним и подключили Telegram-бота. Теперь развернём его
так, как вы запускали бы его на сервере — с помощью Docker Compose.»

### Сцена 2 — Архитектура (0:25–1:15)

ON SCREEN: простая схема —
```
[ argo-brain ]  <-- Unix socket -->  [ argo-core ]  --> port 8000
   Python brain     общий том            Rust gateway
```

NARRATION: «В Docker-конфигурации ARGO два сервиса. argo-brain — это
Python-мозг, он запускает цикл агента через Unix-сокет. argo-core — это
Rust-шлюз, он предоставляет HTTP API на порту 8000 и общается с мозгом
через тот же сокет. Файл compose связывает их двумя общими томами: один
для сокета, другой для постоянных данных.»

### Сцена 3 — Знакомство с файлом compose (1:15–2:15)

ON SCREEN: открытый `docker-compose.yml`, медленная прокрутка.

NARRATION: «ARGO поставляется с готовым docker-compose.yml в корне
репозитория. Обратите внимание: оба сервиса используют одно значение
ARGO_IPC_SOCKET — это общий путь к сокету. Том argo-data хранит вашу базу
данных SQLite и навыки, поэтому данные переживают перезапуски. А у
argo-core есть проверка состояния, обращающаяся к эндпоинту
слеш-api-слеш-health.»

### Сцена 4 — Сборка и запуск (2:15–3:25)

ON SCREEN:
```
docker compose up -d --build
```

NARRATION: «Из корня репозитория выполните docker compose up. Флаг
дефис-d запускает всё в фоне, дефис-дефис-build при первом запуске
собирает образы. Docker собирает образы Python и Rust и запускает два
контейнера.»

ON SCREEN:
```
docker compose ps
```

NARRATION: «Проверьте состояние командой docker compose ps. Дождитесь,
пока argo-core покажет статус healthy — это значит, что проверка
состояния прошла.»

### Сцена 5 — Проверка API (3:25–4:15)

ON SCREEN:
```
curl http://localhost:8000/api/health
```

NARRATION: «Теперь проверьте с хоста. Запрос к эндпоинту health должен
вернуть ответ OK. Шлюз также предоставляет слеш-api-слеш-chat для
разговоров и слеш-metrics для Prometheus. Ваш экземпляр ARGO теперь
работает как полноценный самостоятельно размещённый сервис.»

### Сцена 6 — Логи и управление (4:15–4:45)

ON SCREEN:
```
docker compose logs -f
docker compose down
```

NARRATION: «Используйте docker compose logs, чтобы следить за тем, что
делает каждый сервис, и docker compose down, чтобы всё остановить. Ваши
данные останутся в сохранности в именованном томе и будут готовы к
следующему запуску.»

### Сцена 7 — Заключение и призыв к действию (4:45–5:05)

ON SCREEN: логотип ARGO, адрес GitHub, надпись «Спасибо за просмотр».

NARRATION: «Вот ARGO, развёрнутый с Docker Compose — полностью на вашей
собственной инфраструктуре. ARGO — открытый и под лицензией MIT, он в
стадии альфа на пути к общему релизу, так что сейчас лучшее время
присоединиться. Поставьте звезду репозиторию, заводите issue, а если вы
владеете каким-либо из центральноазиатских языков, ваши правки особенно
приветствуются. Спасибо, что посмотрели всю серию.»
