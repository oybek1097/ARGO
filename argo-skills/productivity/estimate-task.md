# Estimate a Task Realistically

1. Break the task down until each piece is clearly understood.
2. Estimate each piece; vague pieces signal a needed spike.
3. Add explicit buffer for testing, review, and integration.
4. Give a range, not a single number, and state the assumptions.
5. Re-estimate when scope or understanding changes.
