# Break Down a Project Into Tasks

1. State the goal and the definition of done.
2. Decompose into milestones, then into tasks of under one day each.
3. Map dependencies and identify the critical path.
4. Flag risks and unknowns that need a spike first.
5. Sequence the work and set check-in points.
