# Document an Unfamiliar Codebase

1. Map the entry points and the top-level module structure.
2. Trace one core user flow end to end and note the key components.
3. Document the build, test, and run commands first — they unblock others.
4. Add module-level docstrings explaining responsibility, not mechanics.
5. Write a README with setup, architecture overview, and a contribution guide.
