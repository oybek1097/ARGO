# Threat-Model a New Feature

1. Diagram the data flow and trust boundaries.
2. Enumerate threats per boundary (spoofing, tampering, disclosure, DoS).
3. Rate each by likelihood and impact.
4. Define a mitigation or an accepted-risk note for each.
5. Re-review when the design or data sensitivity changes.
