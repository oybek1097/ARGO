# Write a Documentation Page

1. Identify the reader's goal and what they already know.
2. Open with what the page covers and the prerequisites.
3. Give numbered, copy-pasteable steps with expected output.
4. Add a troubleshooting section for common failure modes.
5. End with next steps and links to related pages.
