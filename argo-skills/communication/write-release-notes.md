# Write Release Notes

1. Group changes: new features, improvements, fixes, breaking changes.
2. Write each entry from the user's perspective, not the commit's.
3. Call out breaking changes prominently with a migration note.
4. Link to docs or issues for anything non-obvious.
5. Keep entries short — one line each where possible.
