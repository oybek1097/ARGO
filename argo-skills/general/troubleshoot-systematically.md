# Troubleshoot a Problem Systematically

1. Reproduce the problem reliably and define 'expected vs actual'.
2. Note what recently changed — changes cause most failures.
3. Bisect: halve the search space with each test.
4. Form one hypothesis at a time and test it directly.
5. Confirm the fix addresses the root cause, then verify nothing else broke.
